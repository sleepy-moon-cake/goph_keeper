# GophKeeper (Клиент-серверное хранилище секретов)

GophKeeper — это надежная многоплатформенная клиент-серверная система для безопасного хранения паролей, данных банковских карт, текстовых заметок и бинарных файлов. Защита приватной информации от несанкционированного доступа, компрометации данных и утечек на стороне сервера обеспечивается за счет использования архитектуры **сквозного шифрования (End-to-End Encryption — E2E)**.

## 🔒 Архитектура безопасности

Главный принцип GophKeeper — **Zero-Knowledge (Нулевое разглашение)**. Бэкенд-сервер выступает исключительно в роли «слепого» хранилища байт: он никогда не видит, не передает и не логирует сырые пароли пользователей или ключи шифрования.

### 1. Криптографический конвейер
* **Растяжение ключа (PBKDF2)**: Когда пользователь вводит мастер-пароль, клиентское приложение берет статический неизменяемый параметр — имя пользователя (`username`) в качестве соли, и прогоняет их через `pbkdf2.Key`. На выходе в памяти клиента генерируется **32-байтный статический мастер-ключ AES-256**.
* **Шифрование данных (AES-GCM)**: Любые структурированные бизнес-данные (`CardData`, `TextData`, `BinaryData`) сериализуются в JSON, шифруются локально на клиенте с помощью мастер-ключа, и для каждой записи через `crypto/rand` генерируется случайный криптографический **12-байтный Nonce (соль шифра)**.
* **Хранение в базах данных**: В PostgreSQL (на сервере) и SQLite (в локальном кэше) отправляется и сохраняется только случайный набор зашифрованных байт (`payload`) и инициализирующие векторы (`nonce`).

### 2. Изоляция ключей в памяти (Фоновый агент)
Для удобства пользователя (UX) без потери безопасности система разделена на два независимых процесса, общающихся внутри вашего компьютера по протоколу TCP (`127.0.0.1:9999`):
* **CLI-команды (`add`, `get`, `list`)**: Короткоживущие процессы утилиты, которые при старте мгновенно запрашивают контекст безопасности у локального порта, шифруют/расшифровывают данные и тут же завершают работу.
* **Сервер сессий / Воркер (`start`)**: Запускается в фоновом режиме строго в **оперативной памяти (ОЗУ)** компьютера при успешном логине. Он держит в памяти мастер-ключ и JWT-токен. Эти данные **никогда не записываются на жесткий диск** и бесследно исчезают из памяти при вызове `make stop-client` или выключении ПК.

## ⚙️ Быстрый старт

Управление всеми компонентами распределенной системы автоматизировано с помощью простых команд `make`.

### Требования
* Go 1.26+
* Docker и Docker Compose
* Установленный локально компилятор Protobuf (`protoc`)

### Шаг 1. Запуск серверной инфраструктуры
Откройте основное окно терминала и запустите базу данных, автоматические серверные миграции и сам Go-сервер бэкенда:

```bash
make start
```
*Эта команда сама проверит наличие необходимых папок (`mkdir -p`), поднимет PostgreSQL в Docker, накатит структуру таблиц через `golang-migrate` и запустит сервер.*

### Шаг 2. Инициализация сессии клиента
Откройте **новую вкладку или окно терминала** для работы с консольным клиентом.

1. **Запустите фонового агента в ОЗУ**:
   ```bash
   make start-client
   ```
2. **Зарегистрируйте аккаунт** (если запускаете систему впервые):
   ```bash
   make register
   ```
3. **Авторизуйтесь в системе** (для получения JWT-токена):
   ```bash
   make login
   ```
*Мастер-ключ сгенерируется из вашего пароля и логина и останется заперт в оперативной памяти фонового воркера.*

## 💻 Использование CLI-клиента

Когда фоновый агент запущен, вы можете выполнять команды мгновенно через быструю обертку `make cli args="..."` без постоянного ввода паролей:

### 1. Сохранить секретную текстовую заметку
```bash
make cli args="add --text --name \"Код восстановления Яндекс Почты\""
```

### 2. Прочитать и расшифровать секрет
```bash
make cli args="get --name \"Код восстановления Яндекс Почты\""
```

### 3. Вывести список всех названий секретов
Метод запрашивает «сухие» метаданные из репозитория бэкенда, не требуя доступа к ключу шифрования, что обеспечивает максимальную скорость работы:
```bash
make list
```
